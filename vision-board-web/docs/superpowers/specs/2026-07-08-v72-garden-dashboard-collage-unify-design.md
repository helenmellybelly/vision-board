## C. 대시보드 = 토리의 정원 맵

**원칙:** 부담의 원인은 "6개가 보이는 것"이 아니라 "6개가 전부 행동을 요구하는 것".
전체 지형은 항상 보여주되(6개 안내 + 한눈 파악), 말을 거는 칸은 추천 1개뿐.

### C1. MiniBoardPreview 시각 언어 개편
`components/MiniBoardPreview.tsx`

- **미시작 칸:** 빈 사진 슬롯(회색) → 은은한 섹션색 배경 + 🌱 씨앗 + 섹션명. "미완료 과제"가 아니라 "심을 자리"로.

- **완성 칸:** 대표 사진 + ✓ 뱃지 (질문+사진 다 했다는 표시)
- **추천 칸:** 토리 얼굴(`tori-profile-bust.png`) 마커가 칸 모서리에 앉음 + 은은한 강조 — 게임 맵의 "현재 스테이지" 문법. 기존 `nextSectionId` prop 재사용.
- 접근성: 각 셀 `aria-label`에 상태 문구 유지·갱신

## D. 배경화면 = collage 한 화면 통합

`app/collage/page.tsx` + 대시보드/피니시 진입점

### D1. 뷰 구조
- `choose` 뷰 **제거**. 진입하면 보드가 바로 보임.
- 상단 세그먼트 토글 **[보드 | 📱 폰 | 🖥️ PC]** — `view` 상태는 `'board'|'phone'|'desktop'` 3값으로 축소.
- `picking` 스텝 제거: 폰/PC 토글 시 사이즈 선택(`DevicePresetPicker`)을 **같은 화면 인라인 칩/리스트**로 렌더. 프리셋 미선택이면 보드 위에 사이즈 선택부터, 선택하면 즉시 그 비율로 미리보기 반영. 사이즈는 선택 후에도 칩으로 화면에 남아 바로 변경 가능.
- 비율 다른 프리셋 변경 시 기존 `confirmReseed` 경고 플로우 유지 (배치 소실 방지 확인).
- 레이아웃 저장 키(`collageLayouts` vs `collageDeviceLayouts`)는 **스키마 불변** — 뷰 상태머신만 정리, additive 원칙 유지.
- 코치마크: `view==='board'` 조건이었던 것을 통합 화면 첫 진입 기준으로 조정 (딥링크 진입 시에도 1회 노출).

### D2. 진입점 정리
- 대시보드: "🖼️ 내 비전보드 보기" 1개 (사진 1장부터 노출, 기존 hasAnyImage 조건 유지)
- `?device=phone|desktop` 딥링크는 유지 — 통합 화면에서 해당 토글이 활성화된 채 열림 (/finish 등 외부 진입용)
- `/finish`의 "폰 배경화면으로 만들기" → `/collage?device=phone`으로 목적지 일치화 (현재는 둘 다 맨 collage로 가서 카피와 도착지 불일치)
