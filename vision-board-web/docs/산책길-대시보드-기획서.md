# 산책길 대시보드 테마 기획서 (v7.3 — 설계만, 구현 보류)

> 상태: **검토 대기**. 사용자 확인 후 다음 라운드에서 구현 여부 결정.
> v7.4 반영: "피어나/피었어" 개화 카피는 v7.4에서 선제 교체됨(중립 진행 카피로). 산책길 테마 자체의 착수 여부는 여전히 별도 결정 사항 (`리플래닝-v74.md` §6).
> 관련 피드백(2026-07-08): "갑자기 '정원' 컨셉이 튀어나오는 대신, 정원사 토리 캐릭터와 연결된 '산책길' 테마는 어떨까. 토리가 길을 따라 움직이며 성장하는 형식을 검토해 달라. '피었어' 같은 문구도 어색하다."

## 1. 배경·문제

- 현재 대시보드는 6칸 폴라로이드 **정원 맵**(`MiniBoardPreview`): 빈 칸 = 씨앗🌱/새싹🌿, 사진이 담기면 "피었어" 카운트.
- 문제 인식:
  - 온보딩(도토리를 심는다) → 대시보드(정원이 핀다) 서사가 **은유 점프**로 느껴짐. 도토리는 나무가 되는 씨앗인데 대시보드에선 꽃이 "핀다"로 어휘가 어긋남.
  - "1/6 피었어" 같은 문구가 어색하다는 직접 피드백.
  - 6칸 그리드는 정적 — 진행감(여정)이 안 보인다.
- 토리는 "꿈의 **정원사**"로 소개되므로, 정원을 버리는 게 아니라 **정원 사이를 걷는 산책길**로 통합하면 캐릭터와 자연스럽게 연결된다.

## 2. 컨셉 정의 — "토리와 걷는 산책길"

- **길 = 진행**: 온보딩에서 심은 도토리(출발점)에서 시작해, 6개 스테이션(섹션)을 지나 완성된 비전보드(참나무 언덕)에 도착하는 S자 산책길.
- **토리 = 현재 위치**: 토리 마커가 다음 추천 스테이션 앞에 서 있다 (v7.2의 "게임 맵 현재 스테이지 문법"을 길 위로 확장).
- **스테이션 = 섹션**: 각 스테이션은 그 섹션의 폴라로이드 미니 프레임. 상태별 시각:
  - `not_started` — 안개 낀 빈 표지판 + 🌰
  - `in_progress` — 표지판에 사진 1장 살짝 보임 + 📷
  - `text_complete` — 표지판에 글귀 마크 ✍️
  - `completed` — 프레임 완성 + 작은 나무 🌳
- **어휘 교체**: "피었어" → "지났어/걸었어" 계열. 예: `2/6 스테이션을 지났어 🐿️`, 완주 시 `길 끝에 도착! 배경화면으로 만들어보자`.

## 3. 화면 구조안

```
[헤더: 토리 아바타 + "{이름}의 비전 산책길"]
[산책길 맵 — 세로 S자 경로]
  출발: 심은 도토리 🌰 (온보딩 회상)
  스테이션 1~6 (섹션 순서 or 추천 순서)
  도착: 참나무 언덕 → /collage
[추천 카드: "토리가 {스테이션} 앞에서 기다려"]
[하단 액션: 사진 바로 담기 / 내 비전보드 보기]
```

- **구현 단위**: `MiniBoardPreview` 대체가 아니라 **병행 컴포넌트** `WalkPathMap.tsx` 신설 → 대시보드에서만 교체. `MiniBoardPreview`는 collage/review/finish 등 공용 3곳에서 계속 사용 (보드 프리뷰 본연의 역할).
- 스테이션 탭 = 기존 `handleSelectSection` 재사용 (양경로 시트 포함).
