# 피드백 라운드 v7.7 — 실사용 피드백 2차분 기획

> 상태: **확정** (2026-07-24 오너 결정 4건 반영). 기준 기획서 `리플래닝-v74.md` §5 로드맵의 v7.7 행이 이 문서를 가리킨다.
> 원칙: 이 서비스는 "빨리 끝내는" 서비스가 아니라 **충분한 시간을 들여 차분하게 만들어가는 과정**이다(리플래닝-v74 §0). 이번 라운드의 넛지·지속성 장치는 전부 이 원칙의 표현이다.

## 0. 피드백 원문 요약 (9항목)

1. 완료 시트의 보드 프리뷰가 옛날 버전(흰 폴라로이드) — 수정 또는 삭제
2. 상황별 아이콘: 채팅 답변 중에도 카메라 아이콘 노출 → 단계별 분기
3. 상태 아이콘 호버 툴팁
4. 유연한 경로: 나→건강→관계 고정 점선 길 — 의견 요청
5. 정원사 토리 프로필 활용한 보드 제작 가이드 (UX 해치면 제외 가능)
6. "다른 예시 보기" 3→5개 확대
7. 전체 플로우 점검·기획 보완
8. 넛지: "차분히 만들어가는 과정" 가치를 온보딩·대시보드 적재적소에
9. 지속성 유지 장치 상세 기획

## 2. 상태 아이콘 매핑 (v7.7 확정)

설계 논리: 아이콘이 "현재 상태"가 아니라 **"다음 행동"**을 가리킨다. 재방문 시 지도만 봐도 "어디서 뭘 하면 되는지" 즉답. 단일 소스: `lib/stationStatus.ts`.

| status | 사진 | 구 | 신 | title 툴팁 |
|---|---|---|---|---|
| not_started | ✕ | 🌰 | 🌰 | "{섹션} — 아직 시작 전이야. 토리랑 걸어볼까?" |
| not_started | ○ | 📷 | 📷 | "{섹션} — 사진은 담아뒀어. 이야기도 들려줘!" |
| in_progress (채팅 답변 중) | 무관 | 📷 ❌ | **💬** | "{섹션} — 지금 질문에 답하는 중이야" |
| text_complete (사진 담을 차례) | 무관 | ✍️ | **📷** | "{섹션} — 글은 완성! 이제 사진 담을 차례야" |
| completed | — | 🌳 | 🌳 | "{섹션} — 완성! 나무가 자랐어" |

✍️는 퇴역(대화 앱이므로 💬가 정확). aria-label(`${label} — ${STATUS_LABEL[status]}`)은 아이콘과 독립 계약으로 무변경. 미니보드 뱃지도 동일 정합(✓ 완성 / 📷 사진 차례 / 💬 답하는 중).

## 3. 유연한 경로 — 옵션 비교와 채택 근거

| 옵션 | 데이터 | 공수 | 은유 리스크 |
|---|---|---|---|
| (a) 완료순 마커 재배열 | 불요 | 중 | **높음** — 섹션 라벨이 방문할 때마다 지도 위에서 이동("어제는 관계가 2번째, 오늘은 4번째"). 지형 고정 은유 붕괴 |
| **(b) 경로 고정 + 완료 구간 실선 (채택)** | 불요 (status만으로 판정) | 중 | 없음 — 도토리→참나무 축 보존, "걸은 길은 다져진다" 은유 강화 |
| (c) visitedAt 완전 유연 경로 | 섹션별 방문 시각 신규 기록(스키마 v5) + ANCHORS 좌표계 리팩터 | 대 | 중 |

채택 근거: 피드백의 본질은 "순서를 건너뛰어도 어색하지 않게"인데, (b)에서 건너뛴 구간은 점선으로 남아 "아직 안 걸은 길"로 자연스럽게 읽히고 완료 구간은 실선 "걸어온 길"이 된다. 현재 데이터로는 방문 순서가 기록되지 않아 (c)는 스키마 마이그레이션이 필요한 대공사 — 백로그(§6).

구현: `buildSegments()` — ANCHORS 7구간을 독립 SVG path로 분해, 도착 스테이션이 completed면 실선(rgba 0.55)/아니면 점선(rgba 0.28, `1 7`).

## 4. 예시 5세트 — 페르소나 체계 (v7.7 확정)

기존 3세트는 **욕구의 방향** 축: ① 결핍형(없는 걸 채우기) ② 유지형(좋은 상태 지키기) ③ 성장형(더 확장하기). 신규 2세트는 직교 축:

- ④ **전환기형** (삶의 국면 축): 이직·이사·독립·복직 등 리셋 국면. 시점 마일스톤 명사("복직 후 100일", "이사 온 지 석 달"), before→after 서사. 결핍형과 차별점: 없는 걸 채우는 게 아니라 **있던 삶을 갈아타는** 이야기.
- ⑤ **관계·공동체형** (관계 반경 축): 혼자의 성취보다 사람과 함께가 중심. **'누구와'가 주어**, 관계 명사+빈도 숫자("친구랑 주 1회 배드민턴", "한 달에 한 번 우리 집 모임").

카피 규칙 승계(v7.6→v7.7): 세트 = 한 사람의 완결 답변, 구체 명사/숫자 ≥1, want는 `\n` 3줄, 반말 '나' 시점. 검토 후 제외한 축: 회복형(결핍형과 겹침), 도전형(성장형과 겹침). 대상: 6섹션×4질문 = 24배열 × 2세트 = 48카피 (`lib/questions.ts` 단일 소스). 씬 일기 예시는 [산문 일기체, 장면 불릿체] 2세트 유지 — 개수가 아니라 형식 시연이 목적.

## 5. 전체 플로우 점검 결과 (항목 7)

점검 범위: 온보딩 3스텝 → 대시보드 → pathSheet → /section(Q&A) → 씬 브리지 → /scene(일기) → /scenes(사진) → 완료 시트 → /collage → /finish.

**이미 잘 갖춰진 것 (기구현 지속성 장치 인벤토리):**
슬롯 유예(deferredSlots), 세션 분할 카피("한 호흡"), 자동 다음 추천(getRecommendedSection + 완료 시트 CTA), pathSheet 직전 선택 기억+3연속 생략, 재생성 합산 2회 제한, 산책길 진행 은유(도토리→참나무·수풀 장식), 복귀 인사, 온보딩 심기·슬라이더 인터랙션, 시간 기대 카피("첫 스테이션은 5분이면 돼").

**이번에 발견·해소한 마찰:** 미니보드 구버전 잔존(#1), 아이콘-단계 불일치(#2), 진행 누적감 부재(#4 실선으로 해소).

**남은 마찰 (백로그 편성):** §6.
