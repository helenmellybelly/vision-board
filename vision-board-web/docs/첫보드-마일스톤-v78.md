## 1. 설계

### 1-1. 단일 소스: `lib/milestone.ts` (신규)
- `FIRST_BOARD_THRESHOLD = 3` — 임계값의 유일한 정의처.
- `countCompleted(board)` — completed 섹션 수.
- `isStoryStale(board)` — 스토리 작성 후 보드가 더 자랐는지(재작성 넛지 판정). `storyWrittenAtCount`가 없는 기존 데이터는 **항상 false** — 기존 사용자에게 갑자기 넛지가 뜨는 소음 방지.

### 1-2. 스탬프: `storyWrittenAtCount?: number` (BoardData 옵셔널, schemaVersion 4 불변)
- `saveFutureDayStory()`가 저장 시점의 completed 수를 함께 기록. 생성·수동 수정(StoryModal) 모두 갱신 — 어느 쪽이든 "현재 보드를 반영한 최신본"이므로 넛지를 접는 게 맞다.

### 1-3. 게이트 완화 (리플래닝-v74 §2 구현 메모 그대로)
| 게이트 | 변경 |
|---|---|
| ① 대시보드 /review 버튼 | **유지** (6섹션 텍스트 완료) |
| ② 대시보드 /finish 진입 | `completedCount >= 3 && !futureDayStory && !allTextDone` 시 조기 개방 CTA "첫 보드가 열렸어" 추가. allTextDone이면 기존 /review CTA가 흐름을 이끔(버튼 중첩 방지) |
| ③ 콜라주 완성 CTA | `completedCount === 6` → `>= 3`. 라벨 분기: 6/6 "내 비전보드 완성하기"(불변, v75r1 계약) / 부분 "첫 보드 완성하기" |

- `/finish` 페이지 자체 무게이트 유지(기존과 동일).
- `api/story` 프롬프트의 "6개 섹션(나·건강·…)" 하드코딩 제거 → "위 섹션들이" (부분 재료에서도 성립).

### 1-4. 재작성 넛지 ("보드가 자랐네 — 이야기 다시 써줄까?")
- 노출 지점 2곳: 대시보드 CTA(추천 카드 아래), 콜라주 이야기 블록의 "다시 쓰러 가기" 라벨 교체.
- 판정은 `isStoryStale()` 단일 소스. 재작성하면 스탬프가 갱신되어 넛지가 접힌다. 최종 스토리 재작성은 diaryRegenCount(섹션 일기 2회 제한)와 무관 — 기존에도 무제한이던 "다시 써줘" 동선 재사용.

### 1-5. 6/6 상향 목표화 ("완전한 보드")
- 발견 지점: 완료 시트(3번째 완성 직후) "🎉 첫 보드가 열렸어" CTA — 임계값 도달의 보상 회수 지점.
- /finish 부분 상태 적응: 패턴 단계 "먼저 자란 N가지 영역", 완성 단계 타이틀 "첫 비전보드가 완성됐어" + "여섯 그루가 다 자라면 완전한 보드가 돼" 카피.
- walkCaption·완료 시트 "완성! N/6이야"는 불변 (기존 스위트 계약 + 높이 예산 ≤1.2뷰포트).
