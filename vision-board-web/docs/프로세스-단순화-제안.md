## 유저 경험

1. **pathSheet 직전 선택 기억** — ✅ v7.4 R1 구현: 미시작 섹션 탭 시 뜨는 "질문/사진" 양경로 시트에서 직전 선택을 기억해 프리하이라이트, 같은 선택 3회 연속이면 시트 생략하고 직행.
2. **섹션 완료 후 자동 다음 추천** — ✅ v7.4 R1 구현: /scenes 저장 완료 시트 "다음 스테이션으로" 점검·보강 + 세션 분할 카피("오늘은 여기까지도 충분해") 병행.
3. **온보딩 스킵 링크** — ❌ 불채택 (2026-07-23 사용자 결정): 온보딩은 이미 3스텝으로 충분히 축소됨. 스킵 대신 각 스텝의 몰입도를 인터랙션·디자인·카피로 강화하는 방향으로 확정 (v7.4 R1).
4. **일기 재생성 비용 안내** — ✅ v7.4 R1 구현(강화판): 카피 안내를 넘어 재생성 합산 2회 제한 + "직접 수정하기" 유도. LLM 자체도 무료 티어로 전환되어 비용 축은 해소.

## 코드·운영 (기술 부채)

5. **온보딩 카피 외부화**: Step1~3의 인라인 JSX 카피를 `lib/onboardingCopy.ts`로 — 카피 라운드마다 단일 파일 diff (이번 라운드로 카피 수정이 8회째). (보류 — v7.4 온보딩 리워크 이후 재평가)

7. **verify 스위트 위생**: 라이브 스위트 10+개, 어서션 중복. 규칙 제안: **"현행 릴리스 스위트 + 직전 1개"만 라이브**, supersede된 스위트는 `.claude/archive/`로 이동 (아카이브 디렉터리 이미 존재). 이번 v7.3에서 v7r4·v71r3 어서션 다수가 superseded.
