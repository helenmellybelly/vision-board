# 프로세스 단순화·효율 개선 제안 (v7.3)

> "복잡성은 줄이고 효율은 높이는 방향" 피드백에 대한 제안 목록. 각 항목 독립 실행 가능.

## 유저 경험 (다음 라운드 후보)

1. **pathSheet 직전 선택 기억**: 미시작 섹션 탭 시 뜨는 "질문/사진" 양경로 시트에서 직전 선택을 기억해 프리하이라이트, 같은 선택 3회 연속이면 시트 생략하고 직행. (반복 유저의 탭 1회 절약)
2. **섹션 완료 후 자동 다음 추천**: /scenes 저장 완료 시트에서 "다음 스테이션으로" 버튼이 이미 있는지 점검, 없으면 추가 — 대시보드 왕복 없이 연속 진행.
3. **온보딩 스킵 링크**: 재설치/재방문 유저용 "바로 시작하기" (온보딩 3스텝 하단 micro 링크). 데이터 있으면 어차피 대시보드 직행이므로 신규 유저의 재진입 케이스만 해당.
4. **일기 재생성 비용 안내**: "하루 다시 쓰기"가 무제한처럼 보임 — 실제 API 비용 발생. 2회 이후 "직접 수정하기"를 먼저 권하는 카피 검토.

## 코드·운영 (기술 부채)

5. **온보딩 카피 외부화**: Step1~3의 인라인 JSX 카피를 `lib/onboardingCopy.ts`로 — 카피 라운드마다 단일 파일 diff (이번 라운드로 카피 수정이 8회째).
6. **LLM SDK 감사**: package.json에 `@anthropic-ai/sdk`, `@google/generative-ai`, `groq-sdk`, `openai` 4종. 실사용은 openai(/api/story/section 등)·groq(/api/story) 중심으로 보임 — 미사용 SDK 제거로 의존성·설치시간 절감. (제거 전 grep으로 사용처 확정)
7. **verify 스위트 위생**: 라이브 스위트 10+개, 어서션 중복. 규칙 제안: **"현행 릴리스 스위트 + 직전 1개"만 라이브**, supersede된 스위트는 `.claude/archive/`로 이동 (아카이브 디렉터리 이미 존재). 이번 v7.3에서 v7r4·v71r3 어서션 다수가 superseded.
8. **하드코딩 연도·기간 grep 훅**: targetDate가 유저 편집 가능해졌으므로, 커밋 전 `3년|20\d\d년` grep을 verify 스위트에 1케이스로 추가 — 새 하드코딩 유입 방지.

## 분석 (이번 라운드에 기반 마련됨)

9. Vercel Web Analytics 설치 완료 + track() 5종 심음. **커스텀 이벤트는 Pro 플랜 전용** — Hobby면 페이지뷰만 수집되고 track은 무해한 no-op. 플랜 확인 후 필요 시 업그레이드 또는 이벤트만 자체 API(/api/track → 로그)로 대체 검토.
